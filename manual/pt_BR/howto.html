Como executar a AppImage do fre:ac?

Após baixar a Appmage do fre:ac, torne-a executável. No gerenciador gráfico de arquivos, isso pode ser feito clicando com o botão direito do mouse no AppImage e clicando em Propriedades. Na caixa de diálogo a seguir, mude para a guia Permissões e marque a opção Executar como aplicativo. As etiquetas individuais podem ter nomes um pouco diferentes dependendo da sua distro e do seu ambiente da área de trabalho.

Ou você também pode definir o arquivo como executável direto no terminal. Vá até o diretório onde o AppImage está e digite:

chmod a+x freac-v1.1.1-linux-x86_64.AppImage

Depois rode o AppImage com o comando abaixo:

./freac-v1.1.1-linux-x86_64.AppImage

Também é possível rodar a interface de linha de comando do fre:ac e o aplicativo smooth Translator incluso usando o AppImage:

  • freaccmd: ./freac-v1.1.1-linux-x86_64.AppImage freaccmd
  • smooth Translator: ./freac-v1.1.1-linux-x86_64.AppImage translator

Caso precise acessar o conteúdo do AppImage fre:ac, rode ele com a opção --appimage-extract:

./freac-v1.1.1-linux-x86_64.AppImage --appimage-extract

Após rodar o comando, o conteúdo do arquivo AppImage vai estar na pasta squashfs-root.

Como mudar o tamanho da fonte de fre:ac?

Caso as letras da interface estejam muito pequenas, você pode ativar o dimensionamento com a opção --scale:n.n. Defina n.n para o fator de escala desejado. Por exemplo, para que o texto 25% maiores, execute o fre:ac assim:

freac --scale:1.25

Como usar os padrões para gerar os nomes dos arquivos no fre:ac?

O padrão para a nomenclatura dos arquivos gerados pode ser configurado na página do Codificador. É possível usar as barras invertidas para definir os diretórios e os marcadores de posição para se referir as metainformações. Os marcadores disponíveis são:

  • <artist> - O cantor
  • <title> - O título da música
  • <album> - O título do álbum
  • <disc> - O número do disco (caso tenha mais de um)
  • <track> - O número da faixa
  • <genre> - O gênero da música
  • <albumartist> - O artista do álbum
  • <composer> - O compositor da música
  • <conductor> - O condutor
  • <ano> - O ano em que a música foi publicada
  • <discid> - A identificação CDDB/freedb do disco
  • <currentdate> - O dia de hoje no formato aaaammdd
  • <currenttime> - A hora atual no formato hhmm

Os marcadores para <disc> e para <track> podem ser alterados para <disc(n)> e <track(n)> onde a parte (n) define o tamanho mínimo do disco ou o número da faixa. Os zeros serão adicionais caso o número real seja pequeno demais. O n pode ser entre 1 a 4 onde 1 é o padrão para o disco e 2 é o padrão para a quantidade das trilhas caso a parte (n) seja omitida.

Os marcadores de posição acima se refere às informações da música e do álbum. O marcador a seguir define o tipo do arquivo gerado:

  • <filetype> - O tipo do arquivo gerado

Um caso muito comum para o uso deste marcador é quando você converte toda hora para formatos diferentes (de FLAC para arquivamento e M4A para o seu tocador de música portátil por exemplo) e gostaria de manter os arquivos codificados em diretórios diferentes.

Além do que foi dito acima, há dois marcadores que se referem ao nome original do arquivo que está sendo codificado no fre:ac:

  • <filename> - O nome do arquivo que será recebido
  • <directory> - A pasta onde o arquivo recebido pode ser encontrado

O padrão <directory> é especial, pois é possível alterar o seu significado usando dois argumentos numéricos. Você pode usá-lo como <directory+n> ou <directory+n (m)>. Nessas expressões o n define a quantidade dos caminhos dos componentes que devem ser retirados desde o início. O m define a quantidade de componentes que serão usados a partir do que sobrar. A omissão do (m) usará todos os componentes deixados no caminho após a remoção de n componentes a partir do início. <directory> sem qualquer argumento apontará para o caminho completo.

Então como fica? Supondo que você tenha a sua coleção de músicas ordenada desta maneira:

C:\Music\[outro critério]\[artist]\[album]\[track] - [title]

Você pode ter um arquivo chamado C:\Music\Favorites\Starsailor\Silence is Easy\01 - Music was Saved.flac.

Agora, vamos supor que queira converter alguns arquivos de música para uso no seu reprodutor portátil e colocá-los em C:\Encoded\. Para manter seu campo [outro critério], você precisa usar o padrão <directory>. Caso a pasta de destino do codificador esteja definido como C:\Encoded\, o padrão do nome do seu arquivo completo ficará assim:

<directory+1(1)>\<artist> - <album>\<artist> - <album> - <track> - <title>

Isso vai converter o arquivo mencionado anteriormente para C:\Encoded\Favorites\Starsailor - Silence is Easy\Starsailor - Silence is Easy - 01 - Music was Saved.m4a.

Como configurar a pasta destino para uso portátil?

Ao usar o fre:ac a partir de uma unidade portátil como de um HDD externo ou uma unidade USB por exemplo, é preferível que a pasta de destino se ajuste automaticamente às circunstâncias encontradas nos diferentes computadores. As letras das unidades podem ser diferentes no seu drive portátil. O fre:ac é compatível com os seguintes marcadores para a definição da configuração da pasta de destino:

  • <installdrive> - É a letra da unidade onde o fre:ac está instalado (apenas no Windows)
  • <profile> - A pasta do perfil do usuário conectado atualmente

Caso queira que os seus arquivos convertidos vá para a unidade portátil, independentemente da letra da unidade atribuída a ela, use a marcação <installdrive> e configure a pasta de destino como, por exemplo:

<installdrive>\Music Files\

Caso queira que os ser arquivos convertidos vá para a pasta Minhas músicas do usuário conectado atualmente, você pode usar a marcação <perfil> da seguinte maneira:

<profile>\Music\

Nos dois casos, o caminho exibido na parte inferior da janela principal do fre:ac refletirá a pasta real do destino com os marcadores já resolvidos.

Como instalar um codificador/decodificador A ou B?

Aften AC-3 Encoder

Para instalar o codificador Aften AC-3 por exemplo, siga estas etapas:

  • Baixe o pacote ZIP binário Aften Win32 em sourceforge.net.
  • Copie o arquivo aften.exe do pacote ZIP para a subpasta codecs/cmdline no diretório onde o seu fre:ac está instalado.
Decodificador Apple Lossless

O decodificador Apple Lossless é originalmente fornecido junto com o fre:ac, assim sendo, você não precisa instalá-lo manualmente.

Caso ainda precise instalar ou atualizar o decodificador manualmente, siga estas etapas:

  • Baixe o pacote genérico de compilação do Win32 em www.rarewares.org.
  • Copie o arquivo alac.exe do pacote ZIP para a subpasta codecs/cmdline no diretório onde o seu fre:ac está instalado.
Codificador DTS Coherent Acoustics

Para instalar o codificador DTS Coherent Audio, siga estas etapas:

  • Baixe o pacote ZIP binário ffdcaenc em GitHub.
  • Copie o arquivo ffdcaenc.exe do pacote ZIP para a subpasta codecs/cmdline no diretório onde o seu fre:ac está instalado..
exhale xHE-AAC Encoder

Para instalar o codificador exhale xHE-AAC, siga estas etapas:

  • Baixe o pacote exhale ZIP em www.rarewares.org.
  • Copie o arquivo exhale.exe do pacote ZIP para a subpasta codecs/cmdline no diretório onde o seu fre:ac está instalado..
Decodificador Extended Module Player (XMP)

Para instalar o decodificador Extended Module Player (XMP), siga estas etapas:

  • Baixe o pacote Extended Module Player 7z em modland.com.
  • Copie os arquivos XMP.exe e LibXMP.dll do pacote 7z para a subpasta codecs/cmdline no diretório de instalação do fre:ac.
Codificador FLACCL OpenCL FLAC

Para instalar o codificador FLACCL OpenCL FLAC, siga estas etapas:

  • Baixe o pacote FLACCL RAR em cue.tools.
  • Crie uma nova pasta chamada flaccl dentro da subpasta codecs/cmdline no diretório onde o seu fre:ac está instalado.
  • Copie todos os arquivos do pacote RAR para a subpasta codecs/cmdline/flaccl no diretório onde o seu fre:ac está instalado.
Codificador Flake FLAC

Para instalar o codificador Flake FLAC, siga estas etapas:

  • Baixe o pacote ZIP binário do Flake Windows em flake-enc.sourceforge.net.
  • Copie o arquivo flake.exe do pacote ZIP para a subpasta codecs/cmdline no diretório onde o seu fre:ac está instalado.
Codificador RealAudio Helix DNA Producer

Para instalar o Codificador RealAudio Helix DNA Producer, siga estas etapas:

  • Crie uma conta na Comunidade Helix em helixcommunity.org. Você precisará dessa conta para fazer o download do aplicativo Helix DNA Producer.
  • Baixe o pacote ZIP do aplicativo de linha de comando Helix DNA Producer do helix-producer.helixcommunity.org.
  • Crie uma nova pasta chamada helix dentro da subpasta codecs/cmdline no diretório onde o seu fre:ac está instalado.
  • Copie o conteúdo da pasta producer do pacote ZIP para a subpasta codecs/cmdline/helix que foi recém-criada no diretório onde o seu fre:ac está instalado. O arquivo producer.exe agora deve estar dentro da pasta helix junto com vários outros arquivos e diretórios do pacote Helix DNA Producer.
Codificador / Decodificador La Lossless Audio

Para instalar o codificador/decodificador de áudio La Lossless, siga estas etapas:

  • Baixe o pacote de instalação do La Windows em www.lossless-audio.com.
  • Execute o executável de configuração e instale o codificador La numa pasta da sua escolha.
  • Copie o arquivo la.exe da pasta de instalação La para a subpasta codecs/cmdline no diretório onde o seu fre:ac está instalado.
  • Observe que o codificador La deve permanecer instalado no seu sistema para que ele possa ser usado com o fre:ac.
Codificador de linha de comando LAME MP3

How to add a new translation to fre:ac?

Use o utilitário smooth Translator que acompanha o fre:ac e abra um arquivo de tradução já existente (encontrado na pasta freac/lang). Em seguida, altere as configurações do idioma para refletir a tradução que você quer criar e comece a traduzir as sequências individuais dos textos. Salve a sua tradução como um novo arquivo no formato freac_??.Xml (substitua o ?? pelo código do seu idioma), reinicie o fre:ac e tente selecionar a nova tradução nas configurações.

Encaminhe a sua tradução para translations@freac.org, para que ela possa ser incluída na distribuição oficial.

Como compilar fre:ac a partir do código-fonte?

Para compilar o fre:ac a partir do código-fonte, é preciso baixar a versão mais recente do fre:ac Component Development Kit (CDK) no site freac.org primeiro.

Você também vai precisar do código fonte mais atual do Smooth Class Library em smooth-project.org e claro, do pacote de código-fonte do fre:ac ou do código fonte oficial do fre:ac no Git.

  • extraia o conteúdo do freac-cdk-1.0.x-windows.zip em qualquer lugar
  • crie uma estrutura de pastas como esta:
    /cdk (contém a pasta system e o cdk.bat)
    /cdk/freac (renomeie a pasta do pacote freac-1.1.x.tar.gz)
    /cdk/smooth (renomeie a pasta do pacote smooth-0.9.x.tar.gz)
  • execute o cdk.bat
  • digite: make && cd smooth && make && cd ../freac && make

Como excluir o fre:ac do meu computador?

Todas as versões mais recentes do fre:ac podem ser removidas através do painel Aplicativos nas Configurações do Windows. Localize fre:ac na lista de aplicativos instalados e selecione Desinstalar para excluir.

Caso você não consiga encontrar o fre:ac lá, basta excluir a pasta do fre:ac. O fre:ac não cria nenhuma entrada no registro do Windows nem armazena qualquer componente ativo fora da sua pasta de instalação.
