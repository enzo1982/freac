Как запустить fre:ac с помощью AppImage?

После загрузки файла AppImage fre:ac, вам необходимо сделать его исполняемым. В графическом файл-менеджере это обычно может быть сделано при помощи клика правой кнопкой мыши на файле AppImage и выборе Свойства. В споявившемся окне переключитесь на вкладку Разрешения и проверьте настройку Разрешить выполнение. Отдельные пункты могут называться по-другому в зависимости от дистрибутива и окружения рабочего стола.

В качестве альтернативы, вы можете установить атрибут исполнение при помощи терминала. Перейдите в папку, содержащую загруженный файл AppImage и введите:

chmod a+x freac-v1.1.1-linux-x86_64.AppImage

После этого просто запустите файл AppImage при помощи:

./freac-v1.1.1-linux-x86_64.AppImage

Кроме того есть возможность запускать интерфейс командной строки fre:ac и имеющееся приложение Переводчик при помощи AppImage:

  • командная строка: ./freac-v1.1.1-linux-x86_64.AppImage freaccmd
  • Переводчик: ./freac-v1.1.1-linux-x86_64.AppImage translator

Если вам нужен доступ к содержимому файла AppImage fre:ac, вы можете получить его при помощи аргумента --appimage-extract:

./freac-v1.1.1-linux-x86_64.AppImage --appimage-extract

После выполнения этой команды вы сможете найти содержимое файла AppImage в папке squashfs-root.

Как изменить размер шрифта в fre:ac?

Если размер интерфейса fre:ac, установленный по умолчанию слишком мал, вы можете включить масштабирование при помощи опции --scale:n.n. Установите n.n на величину, которая вам нужна. Например, чтобы увеличить шрифт и элементы интерфейса на 25%, запустите fre:ac вот так:

freac --scale:1.25

Как использовать шаблоны имен выходных файлов в fre:ac?

Шаблоны имен выходных файлов могут быть настроены на странице Кодек диалога параметров. Вы можете использовать обратную наклонную черту для задания папок и элементов метаданных. Возможны следующие элеиенты:

  • <artist> - Исполнитель
  • <title> - Название песни
  • <album> - Название альбома
  • <disc> - Номер диска
  • <track> - Номер песни
  • <genre> - Жанр песни

  • <albumartist> - Исполнитель альбома

  • <composer> - Композитор песни
  • <conductor> - Дирижер

  • <year> - Год публикации песни

  • <discid> - Идентификатор диска в CDDB/freedb

  • <currentdate> - Текущая дата в формате yyyymmdd
  • <currenttime> - Текущее время в формате hhmm

Элементы <disc> и <track> могут быть изменены следующим образом: <disc(n)> и <track(n)>, где элемент (n) определяет минимальную длину номера диска или песни. Если номер слишком мал, он будет дополнен предшествующими нулями. n может быть в диапазоне от 1 до 4, где 1 - значение по умолчанию для номера диска и 2 - значение по умолчанию для номера песни если (n) не указано.

Предыдущие элементы относились к информации об альбоме и песне. Следующие же обращаются к типу выходного файла:

  • <filetype> - Тип выходного файла
Эти элементы могут использоваться если вы часто кодируете в разные форматы (т.е. FLAC для архива и M4A для вашего плеера) и хотите хранить файлы в раздельных ветках дерева папок.

В дополнение к вышеуказанному есть два элемента, ссылающиеся на имя файла, который будет перекодирован в fre:ac:

  • <filename> - Имя входного файла
  • <directory> - Папка, где находится входной файл

Элемент шаблона <directory> особенный, поскольку вы можете изменять его значение, используя два численных аргумента. Вы можете использовать его как <directory+n> или <directory+n(m)>. В этих выражениях n определяет количество элементов пути, которые будут отделены от начала. m определяет количество компонентов из оставшихся, которые будут использованы. Отсутсвие части (m) означает использование всех компонентов, оставшихся после отделения n компонентов от начала. <directory> без каких либо аргкментов возвращает полный путь.

Как же использовать это на практике? Предположим ваша музыкальная коллекция упорядочена следующим образом:

C:\Music\[other criteria]\[artist]\[album]\[track] - [title]

У вас может быть файл, называющийся C:\Music\Favorites\Starsailor\Silence is Easy\01 - Music was Saved.flac.

Теперь, скажем, вы захотели перекодировать некоторое количество песен для плеера и положить их в C:\Encoded\. Для сохранения поля [other criteria] вам необходимо использовать шаблон <directory>. Если выход кодека установлен в C:\Encoded\, полный шаблон имени файла может выглядеть так:

<directory+1(1)>\<artist> - <album>\<artist> - <album> - <track> - <title>

Упомянутый выше файл будет закодирован в C:\Encoded\Favorites\Starsailor - Silence is Easy\Starsailor - Silence is Easy - 01 - Music was Saved.m4a.

Как настроить выходную папку для портативного использования?

При использовании free::ac с портативного накопителя, такого как внешний HDD или USB диск, желательно, чтобв выходная папка была автоматически настроена с учетом особенностей разных компьютеров. Например это могут быть разные буквы дисков. fre:ac поддерживает следующие элементы для настройки выходной папки:

  • <installdrive> - Буква диска, где установлен fre:ac (только на Windows)
  • <profile> - Папка текущего пользователя

Если вы хотите, чтобы выходные файлы записывались на съемный диск независимо от назначенной ему буквы, используйте элемент <installdrive> и настройте выходную папку похожим образом:

<installdrive>\Music Files\

Если же, напротив, вы хотите, чтобы выходные файлы были помещены в папку My Music в профиле текущего пользователя, выможете использовать элемент <profile> следующим образом:

<profile>\Music\

В обоих случаях выходной путь отображаемый в нижней части главного окна fre:ac будет показывать текущий выходной путь, в котором элементы заменены актуальными значениями.

Как установить кодек/декодер xy?

Кодек Aften AC-3

Для установки кодека Aften AC-3 для использования в fre:ac, выполните следующие шаги:

  • Скачайте zip архив Aften Win32 из sourceforge.net.
  • Скопируйте файл aften.exe из ZIP архива в подпапку codecs/cmdline вашего пути установки fre:ac.
Декодер Apple Lossless

Декодер Apple Lossless обычно поставляется с fre:ac и вам нет необходимости устанавливать его вручную.

Если вам все же необходимо установить или обновить декодер Apple Lossless вручную, выполните следующие шаги:

  • Скачайте Win32 ZIP архив из www.rarewares.org.
  • Скопируйте файл alac.exe из ZIP архива в подпапку codecs/cmdline вашего пути установки fre:ac.

Как добавить новый перевод в fre:ac?

Воспользуйтесь, пожалуйста, утилитой Translator, входящей в поставку fre:ac и откройте существующий файл перевода (можно найтив папке freac/lang). Затем измените настройки языка на необходимый для перевода, который вы хотите добавить и начните переводить отдельные строки текста. Сохраните ваш перевод как новый файл в соответствии с шаблоном freac_??.xml (замените ?? на код своего языка), перезапустите fre:ac и попробуйте выбрать новый перевод в окне параметров.

Пожалуйста, пришлите свой перевод на translations@freac.org и он будет включен в официальную поставку.

Как собрать fre:ac из исходников?

Для того, чтобы собрать fre:ac из исходников вам необходимо сперва скачать последнюю версию fre:ac Component Development Kit (CDK) из freac.org.

Также вам понадобятся исходники последней smooth Class Library из smooth-project.org и, конечно, архив исходного кода fre:ac или исходники fre:ac взятые с Git.

  • распакуйте содержимое freac-cdk-1.0.x-windows.zip куда захотите
  • создайте структуру папок:
    /cdk (содержащую папки system и cdk.bat)
    /cdk/freac (переименуйте папку, извлеченную из архива freac-1.1.x.tar.gz)
    /cdk/smooth (переименуйте папку, извлеченную из архива smooth-0.9.x.tar.gz)
  • запустите cdk.bat
  • введите: make && cd smooth && make && cd ../freac && make

Как удалить fre:ac с моего компьютера?

Все последние версии fre:ac могут быть удалены через панель Приложения утилиты Параметры Windows. Найдите запись fre:ac в списке установленных приложений и выберите Удалить.

В случае, если вы не можете найти здесь запись fre:ac, вполне достаточно удалить папку приложения fre:ac. fre:ac не создает записей в реестре Windows и не сохраняет никаких активных компонентов за пределами папки, в которой установлена программа.
